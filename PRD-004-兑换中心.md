## 1. 文档概览

* **产品名称**:HashRace Platform 兑换中心
* **功能名称**:奖励码兑换、后台兑换码批量生成与管理
* **目标**:提供一个安全、统一的奖励码核销渠道,用于支持各类运营活动的奖励发放
* **核心价值**:

    1. **统一入口**:玩家可通过简单操作,将外部奖励(如活动奖励、推广奖励)兑换到账户
    2. **安全高效**:后台灵活配置兑换模式,严格控制兑换次数,杜绝恶意刷奖行为
    3. **数据追踪**:后台实时监控兑换数据,精确追踪奖励码的核销情况和活动效果

## 3. 核心功能需求

### 3.1 兑换流程

| 步骤 | 玩家操作 | 系统响应 |
| --- | --- | --- |
| **进入** | 玩家点击 App 内的"兑换中心"入口 | 进入兑换页面,显示输入框和兑换按钮 |
| **输入** | 玩家输入或粘贴**奖励码** | 兑换码由字母和数字组成(例如:CUNJIE)。**输入时,系统应自动将小写字母转换为大写** |
| **校验** | 玩家点击"立即兑换"按钮 | **A. 校验**:验证奖励码的**有效性、是否已被兑换、是否过期** |
| **结果** | - | **B. 兑换成功**:实时发放奖励,并弹出成功提示,显示获得的具体奖励内容。奖励金直接进入**游戏账户余额**。**C. 兑换失败**:弹出失败提示,显示具体失败原因(如:已兑换、已过期、无效码) |

### 3.2 界面元素 UI/UX

* **输入框**:醒目,支持粘贴,并有清晰的占位符提示(如:"请输入您的奖励兑换码")
* **兑换按钮**:颜色醒目,点击后有明显的加载或状态变化反馈
* **奖励提示**:兑换成功后,以弹窗形式展示奖励内容

## 4. 后台管理需求

管理员后台是兑换码的核心管理部分,需确保灵活性和安全性。

### 4.1 奖励码生成与管理

* **批量生成**:支持批量生成具有**唯一性**的奖励码,并生成 Excel 下载
* **奖励内容配置**:每个批次的奖励码需绑定以下信息:

    * **奖励类型及数量(列表)**:(例:泥码/红利、报名券、充值送金券等)
    * **有效期**:设置奖励码的生效和截止日期
    * **备注**:标记奖励码的来源(例:S3 合伙人活动、周年庆活动)
    * **兑换码类型**:
    
        * **限兑一次**(该兑换码未被任何用户使用过,**同批次限兑一次**)
        * **每人限兑一次**(该玩家的 ID 未兑换过该**批次**兑换码)
        
    
* **查询与导出**:后台可根据批次、状态、渠道、有效期等条件查询和导出奖励码

### 4.2 兑换记录

查看所有玩家的兑换行为,包括:

* 兑换时间
* 兑换玩家 ID
* 奖励码
* 兑换状态
