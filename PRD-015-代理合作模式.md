# PRD-015:代理合作模式

## 需求概览：为代理、合伙人提供多种合作模式

| # | 功能 | 说明 |
| --- | --- | --- |
| 1 | **三种合作模式** | 正常代理（默认、主推）/ 负盈利分成 / 拉新人头奖励，同一用户任选其一 |
| 2 | **模式变更走运营** | 不提供自助切换；用户联系运营申请，运营后台变更类型并留记录 |
| 3 | **拉新阶梯奖励** | 按结算周期内有效用户数阶梯计价（如 5\~10 人 600/人，10\~30 人 700/人） |
| 4 | **有效用户定义** | 后台可配（注册 + 首充 + 流水 + 时间窗口） |
| 5 | **人工 + AI 审核** | 人头奖励为现金，刷量动机强，逐条审核后发放 |

**背景：** 平台已有面向普通玩家的首页拉新活动（奖励现金、1 倍流水、无上限）。本需求面向代理：把"专职拉新""专吃负盈利"的合作方式产品化，与现有等级返佣体系并列为三种模式供选择，并通过审核控制刷量风险。

---

## 1. 三种合作模式（三选一）

| 模式 | 收益来源 | 计算规则 |
| --- | --- | --- |
| **一、正常代理（默认、主推）** | 平台活动 + 下级返佣（投注型） | 按 PRD-003-2：场馆级配「负盈利」或「投注额」计算 |
| **二、负盈利分成** | 下级玩家的负盈利 | 负盈利口径按 PRD-003-2，分成比例本模式专属可配（参考值 60%） |
| **三、拉新奖励** | 有效用户人头费 | 阶梯单价 × 有效用户数，见 §3 |

**规则：**

* 同一用户同时只适用一种模式，收益不叠加
* **平台费、资金费对各模式均适用**，费率见 PRD-003-2
* 切换后只按新模式计算：切换前已产生的按原模式结算，切换后的按新模式（与 PRD-003-2 配置生效规则一致）
* 模式二 / 三用户不再参与模式一的下级返佣和平台 C 端活动奖励；其名下新用户也不再触发首页拉新活动（避免同一人头双份成本）

## 2. 模式变更流程

**不提供自助切换。** 用户（含首次申请和后续切换）联系运营提出，运营评估后在后台变更其合作类型，按操作时点生效。

* 前端「合作模式」页仅展示当前模式、各模式规则说明和"联系运营申请"指引
* 后台用户详情页显示当前合作模式 + 变更记录（谁、何时、从什么改到什么）
* 变更操作需二次确认；变更前产生的收益按原模式结算，变更后按新模式

---

## 3. 拉新模式的阶梯奖励

**阶梯表后台可配**，每档 = 人数区间 + 单价，示例：

| 结算周期内有效用户数 | 单价 |
| --- | --- |
| 5 \~ 10 人 | 600 / 人 |
| 10 \~ 30 人 | 700 / 人 |
| 30 人以上 | 待配置 |

* **分段累计计价**：每档单价只适用于该档区间内的人数。例：12 人 = 10 × 600 + 2 × 700 = 7400
* 不足最低档（< 5 人）当期不发放，不滚入下期
* 奖励为**现金**，进现金余额；流水倍数可配，默认 1 倍（可配 0 = 免流水）
* 按结算周期统计与发放（如 15 天一周期），周期结束后 2 天内完成审核（见 §5）

---

## 4. 有效用户定义

复用现有邀请码 / 邀请关系（注册时绑定，一对一，不可更改）。所有条件同时满足才算有效，超时未满足标记失效：

| 条件 | 类型 | 默认值 |
| --- | --- | --- |
| 注册、绑定fa | 必须 | — |
| 首充金额 ≥ X、复冲n次、有效投毒、周期内 | 数字 |  |
| 首充后有效流水 ≥ X | 金额（仅现金场次） | 8888 |
| 无多用户同IP等风控规则 | 风控 | -后台审核时可借助ai分析 |

---

## 5. 审核与防刷

人头奖励是现金且单价高，刷量动机远强于普通活动，审核是本模式的生命线。

**流程：** 周期结束 → 生成待审列表（逐个有效用户） → AI 预审标记风险 → 人工终审 → 通过的计入当期人头数发放。审核须在周期结束后 2 天内完成，超时通过站内信提醒运营（复用现有站内信系统），不自动通过。

**人工审核要点：**

| 检查项 | 可疑信号 |
| --- | --- |
| IP | 代理与新用户同 IP 或同 C 段 |
| 设备 | 同一设备标识注册多个账号 |
| 充值行为 | 最低金额充值后立即达标，无后续活动 |
| 注册模式 | 短时间批量注册，用户名/邮箱格式雷同 |
| 流水模式 | 低风险对赌、互相对打刷流水 |

**AI 规则引擎（V1）：** 自动打分（0\~100），低 / 中 / 高三档，高风险排前优先复核，显示触发原因，规则和权重后台可调：（示例、均可配）

| 规则 | 权重 |
| --- | --- |
| 同 IP 注册 | +30 |
| 同设备标识 | +40 |
| 最低金额充值 | +15 |
| 同一代理 24h 内 ≥ n 个达标 | +25 |
| 新用户达标后 3天无操作 | +20 |
| 80% 以上流水在赔率 < 1.2 | +15 |

> V1 基于现有 IP / 设备标识字段做简化判断；完整设备指纹、同提款地址关联、对赌识别等风控评分模块**单独立项**（后续规划）。

## 6. 场景走查

### A：申请切换 + 拉新结算

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 代理 A（正常模式）联系运营申请拉新模式 | — |
| 2 | 运营评估通过，后台变更为拉新模式 | A 不再拿下级返佣 |
| 3 | 当期 A 邀请 12 人，其中 8 人达标 | 进入待审列表 |
| 4 | AI 标记 1 人高风险 → 人工拒绝，7 人通过 | 7 人落在 5\~10 档 |
| 5 | 发放 7 × 600 = 4200 现金 | 1 倍流水 |

### B：刷量拦截

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 拉新代理 C 24h 内 8 人达标 | — |
| 2 | AI 标记高风险（85 分） | 同 IP +30、批量 +25、最低充值 +15、无后续 +20 |
| 3 | 人工核查同设备 → 全部拒绝 | 当期人头数 0，不发放 |

### C：切换生效时间点

| 步骤 | 操作 | 说明 |
| --- | --- | --- |
| 1 | 代理 B（负盈利模式）月中联系运营申请切回正常模式 | — |
| 2 | 运营评估后在后台变更，即时生效 | 变更前的下级盈亏按负盈利模式结算 |
| 3 | 变更后的下级投注按正常返佣计算 | 两段各算各的 |

---

## 7. 边界情况

| 场景 | 处理 |
| --- | --- |
| 申请期间（运营未操作前）的收益 | 按原模式计算，变更操作时点才切换 |
| 拉新代理被封号 | 待发放奖励取消；已发放按正常规则处理 |
| 新用户被封号 | 未审核的不计入人头；已发放不追回 |
| 阶梯表配置调整 | 下个结算周期生效，不追溯 |
| 同一新用户多次充值 | 只看首充是否达标 |
| 审核超时 | 周期结束后 2 天内审完，超时站内信提醒运营（不自动通过） |

---

## 8. 验收标准

**模式与变更**

- [ ] 用户有且仅有一种合作模式，默认正常代理
- [ ] 无自助切换；运营后台变更类型，二次确认 + 变更记录
- [ ] 变更按操作时点生效，前后各按各的模式结算
- [ ] 模式二/三不参与下级返佣和 C 端活动；名下新用户不触发首页拉新
- [ ] 前端「合作模式」页展示当前模式、规则说明、联系运营指引

**拉新奖励**

- [ ] 阶梯表可配（人数区间 + 单价），分段累计计价；不足最低档不发放、不滚存
- [ ] 有效用户条件可配（首充、流水、时间窗口）；流水仅计现金场次
- [ ] 按周期结算，逐条审核通过后发放现金；流水倍数可配（默认 1，可配 0）
- [ ] 周期结束后 2 天内完成审核，超时站内信提醒

**审核**

- [ ] 待审列表含 AI 风险标记，支持单条/批量操作，拒绝须填原因（内部可见）
- [ ] 规则引擎打分、分级、显示触发原因、高风险优先；规则权重可调
