## 📝 PRD 编写规范

### 文档结构
每个PRD文档应包含以下核心部分：

1. **文档概览** - 产品名称、功能名称、目标
2. **业务流程图** - 使用Mermaid绘制关键流程
3. **功能详细需求** - 按模块拆解功能点
4. **页面原型设计** - UI元素与交互说明
5. **业务规则** - 明确的业务逻辑与计算公式
6. **异常处理与安全策略** - 异常场景处理方案
7. **数据与性能要求** - 数据量、性能指标、监控告警
8. **验收标准（QA）** - 可量化的验收清单

### 命名规范
- 文件命名：`PRD-{编号}-{模块名称}.md`
- 编号三位数，例如：PRD-001、PRD-020
- 使用中文名称，便于识别

### 内容要求
- **详细但不冗余**：覆盖关键业务逻辑，避免过度技术实现细节
- **可落地**：开发人员和测试人员能直接使用
- **结构化**：使用表格、列表、流程图提升可读性
- **示例场景**：通过实际案例说明复杂规则
