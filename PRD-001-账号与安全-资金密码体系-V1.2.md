## 1. 背景与目标

HashRace 是游戏娱乐平台，注册门槛极低（无需实名、无需手机号），当前提现无二次校验。本需求在不大幅增加操作摩擦的前提下，为提现行为加一层资金密码保护。

**核心设计原则**：

* 简洁有效：流程步骤最少化，只在必要节点拦截
* 不强制 KYC：所有找回和验证路径均不依赖身份证/人脸识别
* 绑定作为安全锚：通过强制绑定邮箱/手机号/TG（任一），解决盗号无通知渠道的问题
* **三种验证渠道同权**：邮箱、手机号、TG 均可接收验证码与通知（开发已确认 TG 可行），凡需"绑定/验证/找回"的地方三者一视同仁。真正的"裸账户"仅指三者都没绑的账号

## 2. 现状与目标态对比

| 维度 | 当前现状 | 目标态（本 PRD） |
| --- | --- | --- |
| 注册 | 无 OTP，无手机号 | 保持不变 |
| 提现 | 直接提现，无拦截 | 绑定 → 设资金密码 → 输入密码提现 |
| 资金密码 | 已有基础设置/修改能力 | 补充：规则约束、错误计数、锁定、找回 |
| 新设备登录 | 无验证 | 新增两步验证 + 设备信任 |
| 管理后台 | 无资金密码相关能力 | 新增重置、解锁、审计日志 |

## 3. 用户端完整流程

### 3.1 首次提现全路径

```
用户点击「提现」
      │
      ▼
是否已绑定验证渠道？(邮箱/手机/TG 任一)
      │
   ┌──┴──┐
  没绑   已绑
   │     │
   ▼     ▼
走绑定流程   是否已设资金密码？
   │         │
   │     ┌───┴───┐
   │    没设     已设
   │     │       │
   │     ▼       ▼
   │  设资金密码  输入金额 + 资金密码
   │     │       │
   └─────┘       ▼
  (引导回提现页)  提现成功
```

**步骤进度提示**（页面顶部必须显示）：

| 用户情况 | 步骤提示 |
| --- | --- |
| 未绑定 + 未设密码 | 第 1 步：绑定 / 第 2 步：设密码 / 第 3 步：提现 |
| 已绑定 + 未设密码 | 第 1 步：设密码 / 第 2 步：提现 |
| 已绑定 + 已设密码 | 无需进度提示，直接进提现确认页 |

### 3.2 步骤 A — 绑定验证渠道（邮箱/手机/TG）

入口提示：「为了资产安全，提现前请先绑定邮箱、手机号或 TG」

**操作步骤**：

1. 选择渠道：邮箱 / 手机号 / TG（绑其中一个即可）
2. 输入邮箱地址 / 手机号；TG 则按平台既有绑定方式完成
3. **输入登录密码**（确认本人操作，防止借登录态绑他人账号）
4. 点击「获取验证码」→ 收验证码（邮箱、短信或 TG 消息）→ 输入
5. 绑定成功 → **自动进入步骤 B**，无需用户再次点击提现

TG 号现作为有效验证渠道之一，可接收验证码（开发已确认）。绑定 TG 同样满足提现前置条件。

### 3.3 步骤 B — 设置资金密码

| 情况 | 是否需要 OTP |
| --- | --- |
| 刚完成步骤 A 绑定后直接进入本步骤 | **不需要**（刚验过，未满 10 分钟） |
| 已绑定用户，今天第一次来设密码 | **需要**（防借用登录态偷设密码） |

判定口径：距离该用户上次成功 OTP 验证 < 10 分钟，免重发；否则必发一次。

**验证码渠道规则**（适用于所有需要验证码的环节）：绑定了多个渠道（邮箱/手机/TG）时可选择其一，默认优先邮箱，可手动切换；**切换渠道后，原渠道已发送的验证码立即作废**。只绑了一种则直接使用该渠道，不显示切换。

**资金密码规则**：长度 8～20 位；必须同时含字母和数字；不能与登录密码相同；设置时实时显示强度提示。

**设置成功后**：立即可提现（首次设置不触发 24 小时提现锁）；触发通知：绑定渠道 + 站内信。

### 3.4 步骤 C — 提现确认页

1. 填写提现金额、收款地址
2. 输入资金密码
3. 点击「确认提现」→ 验证通过 → 提现成功

**错误处理**：

| 剩余次数 | 前端表现 |
| --- | --- |
| 还剩 3 次及以上 | 提示「密码错误，请重新输入」 |
| 还剩 2 次 | 提示「密码错误，还有 2 次机会，请谨慎操作」 |
| 还剩 1 次 | 提示「密码错误，还有 1 次机会，再错将锁定 24 小时」 |
| 第 5 次错误 | 提示「密码错误次数过多，提现功能已锁定 24 小时」 |

**错误计数规则**：跨设备、跨 IP 累计（防换设备绕过）；**正确输入一次计数立即清零**；锁定期内再次输入不增加计数；锁定到期后计数自动清零。

### 3.5 修改资金密码

入口：安全中心 → 修改资金密码

**前置确认弹窗（必须）**：点击修改入口后先弹出阻塞式确认弹窗——标题「修改资金密码将限制提现 24 小时」；正文「为防止盗号者修改密码后立即转走资产，修改成功后 24 小时内将无法提现。预计解锁时间：XX月XX日 XX:XX」；按钮【取消】（默认聚焦）/【我已了解，继续修改】；点击遮罩不关闭。

**操作步骤**（确认后进入）：发送 OTP 至绑定渠道（必须，不走 10 分钟免验）→ 输入当前资金密码 → 输入新资金密码 → 确认 → 提交。

**修改成功后**：强制 24 小时提现锁；错误计数清零；触发全渠道通知。锁定期内进入提现页展示解锁时间，不展示提现入口。

## 4. 忘记资金密码 — 自助找回

登录密码找回是平台既有能力，不在本 PRD 范围（方案见 HG-462，采用**方案2**）。本章只描述**资金密码**找回。两套密码独立，找回登录密码不影响资金密码。

### 4.1 场景分类

| 用户类型 | 找回方式 |
| --- | --- |
| 已绑定邮箱 / 手机 / TG（任一可收验证码） | 自助找回（见 4.2） |
| 三渠道都没绑（真·裸账户） | 联系客服，走人工核身工单 |

### 4.2 自助找回流程

入口：提现页输入密码 → 「忘记资金密码」链接

1. 系统展示绑定账号（脱敏，如 138\*\*\*\*5678 / abc\*\*\*\*@gmail.com / TG 昵称）
2. 点击「发送验证码」→ 验证码经所选渠道（邮箱/短信/TG）下发
3. 输入验证码
4. 输入新资金密码（同密码规则）→ 确认 → 重置成功

资金密码是用户自设、加密保存，无法"直发"，因此 TG 用户走"验证码发到 TG → 自行设新密码"。

**重置成功后**：强制 24 小时提现锁（防"盗登录 + 盗渠道 → 重置 → 立即提现"攻击链）；触发全渠道通知（含时间和设备信息）；错误计数清零。

### 4.3 裸账户兜底提示

三种渠道都未绑定时，点击「忘记资金密码」展示：

> 「你的账号尚未绑定邮箱、手机号或 TG，无法自助找回。请联系客服提交申请。」

并提供「联系客服」跳转入口。

## 5. 新设备登录验证

### 5.1 触发条件

用户在未曾登录过的设备上成功登录，系统检测到设备 ID 不在该账号信任设备列表中，触发验证。

### 5.2 验证流程

**情况一：绑定了邮箱 / 手机号 —— 两步验证（逐步展示）**

第一步：确认绑定账号——「请输入你绑定手机号/邮箱的中间 4 位数字以确认身份」（例：手机 138\*\*\*\*5678 → 输入 5678）。点击「验证」按钮，后端校验通过后**才进入第二步**。

第二步：验证码验证——系统自动发送验证码至绑定账号，用户输入 → 点击「确认」→ 登录成功。

**情况二：仅绑定 TG —— 单步验证**

TG 无"中间几位"可输，且验证码私发到 TG 本身已能确认本人，简化为单步：系统直接发验证码到 TG，用户输入 → 「确认」→ 登录成功。

同时绑定多个渠道的，默认走邮箱/手机两步，用户也可切换到 TG 接收验证码。

**验证成功后**：当前设备加入信任设备列表，后续在此设备登录跳过验证。

### 5.3 裸账户处理

账号三种渠道都没绑，新设备登录时：直接放行（不弹验证）+ 记录安全日志（设备/IP/时间）+ 站内信提示「检测到新设备登录，如非本人操作请立即联系客服」。提现绑定要求是下一道安全网。

### 5.4 信任设备管理

* 安全中心可查看已信任设备列表（设备名、最近登录时间）
* 可手动移除单个设备的信任状态
* 用户修改登录密码后，**所有信任设备自动清空**

## 6. 提现锁规则汇总

| 触发场景 | 锁定时长 | 用户侧可否提前解除 |
| --- | --- | --- |
| 首次设置资金密码 | **不锁** | — |
| 修改资金密码 | 24 小时 | 否 |
| 连续输错资金密码 5 次 | 24 小时 | 否（等待自动解除） |
| 自助找回后重置资金密码 | 24 小时 | 否 |
| 管理员重置后用户重新设置 | 24 小时 | 否 |

**其他核心规则**：找回**登录密码**不重置资金密码，两套密码互相独立。

## 7. 通知触发规则

| 事件 | 邮箱 | 短信 | TG | 站内信 |
| --- | --- | --- | --- | --- |
| 首次设置资金密码成功 | ✓ | — | ✓ | ✓ |
| 修改资金密码成功 | ✓ | ✓ | ✓ | ✓ |
| 资金密码自助找回重置成功 | ✓ | ✓ | ✓ | ✓ |
| 连续错误触发 5 次锁 | ✓ | ✓ | ✓ | ✓ |
| 管理员重置资金密码 | ✓ | ✓ | ✓ | ✓ |
| 管理员手动解除提现锁 | ✓ | — | ✓ | ✓ |
| 新设备登录（有绑定） | — | — | — | ✓ |
| 新设备登录（裸账户） | — | — | — | ✓ |

**发送原则**：按用户实际绑定的渠道发送（邮箱/短信/TG），未绑定的通道自动跳过；三个外部渠道都没绑的，只发站内信兜底。表中"✓"指"若绑了该渠道则发"。

**通知内容要素**：涉及密码修改、找回重置、触发锁定的通知，须包含操作时间、操作设备信息，并附「如非本人操作，请立即联系客服」指引。

## 8. 管理后台能力

### 8.1 用户安全状态概览

管理员在用户详情页可查看：资金密码状态、提现状态（正常/锁定中+解锁时间）、锁定原因、连续错误次数、最近密码操作时间、信任设备数量。

### 8.2 重置资金密码

**触发条件**：用户联系客服，经客服核身后，管理员操作。**流程**：点击「重置资金密码」→ 二次确认弹窗 → 填操作原因（必填）→ 提交。**效果**：原资金密码清除；错误计数清零；用户下次进提现页引导重新设置；触发全渠道通知。

**冷却规则**：同一账号 **7 天内只能被管理员重置一次**，未到冷却期按钮置灰并显示剩余天数。

### 8.3 手动解除提现锁

**流程**：点击「解除提现锁」→ 二次确认弹窗 → 填操作原因（必填）→ 提交。

**6 小时保护**：同一账号同一类型的锁，手动解除后 6 小时内不得再次手动解锁，防止客服被社工反复绕过。不同类型的锁互不影响。

### 8.4 操作审计日志

所有管理员对资金密码相关操作自动记录：操作时间、操作人、操作类型、被操作用户 ID、操作原因、操作前后状态。日志不可删除，可按时间/管理员/用户 ID 筛选。

### 8.5 待定事项（倾向不做，评审时确认）

| 待定项 | 不做的理由 |
| --- | --- |
| 管理员操作需二次身份校验 | 后台需额外做一套校验体系，成本高 |
| 解锁须关联客服工单号 | 需打通工单系统，关联方式未定 |
| 后台按钮级权限控制（无权限不可见） | 依赖后台权限体系改造，单独排期 |
| 找回登录密码后 24 小时内提现额外验证码 | 与资金密码本身的保护有重叠，先观察 |

## 9. 老用户迁移说明

| 老用户类型 | 触发时机 | 处理方式 |
| --- | --- | --- |
| 已绑定但无资金密码 | 点击提现时 | 引导设密码（步骤 B，跳过步骤 A） |
| 未绑定且无资金密码 | 点击提现时 | 引导绑定 → 设密码（步骤 A + B） |
| 已绑定且已有资金密码 | 无需引导 | 直接进提现确认页 |

不做强制弹窗要求老用户在登录时补绑定，只在提现节点拦截，降低扰动。

## 10. 验收要点

### 用户端

- [ ] 未绑定用户点击提现 → 弹绑定引导，不跳设密码页
- [ ] 绑定流程中需验证登录密码
- [ ] 绑定成功后自动跳到设密码步骤，不需重新点提现
- [ ] 刚完成绑定立刻设密码：不要求再次 OTP；已绑定用户首次设密码必须发 OTP
- [ ] 多步流程页面顶部显示正确的步骤进度
- [ ] 首次设置成功后立即可提现，不进入 24 小时锁
- [ ] 修改入口必须先弹 24 小时锁定确认弹窗，默认聚焦【取消】
- [ ] 修改密码成功后强制 24 小时锁，锁定期内提现入口隐藏并说明解锁时间
- [ ] 密码输对一次后错误计数清零；错误计数跨设备跨 IP 累计；第 3、4 次错误前端预警；第 5 次锁 24h 发全渠道通知
- [ ] 多绑定用户验证码渠道可切换，切换后原渠道验证码作废
- [ ] 绑定渠道支持邮箱/手机/TG 任一，绑 TG 同样可提现
- [ ] 资金密码忘记入口：已绑任一渠道走自助找回，真·裸账户展示「联系客服」
- [ ] TG 用户资金密码找回：验证码发到 TG → 自行设新密码
- [ ] 自助找回成功后触发 24 小时锁 + 全渠道通知
- [ ] 新设备登录（邮箱/手机）：先输入中间 4 位 → 验证通过才自动发码 → 输入完成；两步均有「验证」按钮不自动跳转
- [ ] 新设备登录（仅 TG）：验证码直发 TG，单步输入完成
- [ ] 真·裸账户新设备：直接放行 + 记日志 + 站内信提示
- [ ] 信任设备列表可查可删；修改登录密码后信任设备全部清空

### 管理后台

- [ ] 用户详情页可查看资金密码状态、锁定状态、错误次数
- [ ] 重置资金密码：有二次确认弹窗，必填操作原因
- [ ] 7 天冷却期内按钮置灰并显示剩余天数
- [ ] 手动解锁：有二次确认弹窗，必填操作原因；6 小时内同类型锁不可再次手动解锁
- [ ] 所有操作自动写入审计日志，日志不可删除
