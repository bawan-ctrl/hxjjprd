# PRD-019:合伙人系统 V2

**版本**:V0.1(草稿)

**整理日期**:2026-06-24

**状态：** 草稿

**关联：** 见 §1.1 —— 取代 PRD-007(等级制)/ PRD-014 / PRD-015;沿用 PRD-012(佣金锁)、PRD-013(场馆彩金);玩家侧见 PRD-018

---

## 1. 文档概览

* **产品名称**:HashRace 合伙人系统 V2
* **功能名称**:合伙人 / 高级合伙人身份、点位配置、流水返佣(实时)+ 净输赢分红(月结)、费用扣减(平台费 / 资金费 / 扶持红利 / 活动优惠)、佣金账户与转出
* **目标**:把玩家转化为推广节点,通过邀请下级产生**流水返佣**与**净输赢分红**,同时把下级带来的成本(平台费 / 资金费 / 玩家福利 / 活动优惠)透明计入直属上级,实现可对账、可正可负的代理分润模型

### 1.1 与既有文档的关系

| 文档 | 关系 |
| --- | --- |
| **PRD-007 玩家合伙人系统(V1)** | **本 PRD 取代其「等级制合伙人」模型**(S0-SSS / G1-GGG 等级、按等级返佣比例)。V2 改为**无等级 + 每合伙人独立点位配置**。PRD-007 的等级、周俸禄 / 升级礼等**玩家成长权益已迁移至 [PRD-018 玩家等级系统](PRD-018-玩家等级系统.md)** |
| **PRD-018 玩家等级系统** | 玩家侧等级与权益(周俸禄、升级礼、实时返水等)的权威来源。**本 PRD 的游戏类型口径复用 PRD-018 §5.3**;下级玩家领取的周俸禄 / 升级礼计入直属上级的「扶持红利」(见 §6.3) |
| **PRD-014 代理返佣模型(已废弃)** | **已被本 PRD 取代**(PRD-014 未实施)。原平台费 / 资金费 / 双模式思路在 V2 重做:平台费按供应商配置、净输赢分红、可为负且跨月结转。本 PRD **自包含,不依赖 PRD-014** |
| **PRD-015 代理合作模式(已废弃)** | **已被本 PRD 取代**(PRD-015 未实施)。原多合作模式由 V2「点位配置 + 是否参与净输赢分红」统一表达。本 PRD **自包含,不依赖 PRD-015** |
| **PRD-012 合伙人佣金管理(佣金锁)** | 沿用 `commission_lock`:开启时佣金余额禁止转出现金。V2 定义可转出额的计算(见 §7.4) |
| **PRD-013 场馆彩金与盈利门槛** | 沿用:**场馆彩金场次流水与盈亏全额排除**返佣 / 净输赢基数 |

### 1.2 范围边界

* 本 PRD 描述**代理 / 合伙人侧**的身份、点位、返佣、费用、账户与看板。
* 玩家侧等级、成长值、玩家自身实时返水在 PRD-018,本 PRD 不重复定义,仅在涉及成本归集处引用。
* 金额一律 `decimal`(实现侧 `DECIMAL(20,6)`),比例 `DECIMAL(10,6)`,**禁止 float**。

## 名词定义

| 名词 | 含义 |
| --- | --- |
| **合伙人 / 高级合伙人** | 两种推广身份,**无等级**(§2) |
| **点位** | 合伙人按**游戏类型**配置的流水返点比例(%)(§3) |
| **点差** | 上级点位 − 下级点位;上级据此获得返佣 |
| **管理费** | 上级点位 **≤** 下级点位时,上级改拿下级该笔返佣的 **10%**(平台补贴);**属返佣的一部分** |
| **返佣** | **合伙人侧**收益:实时流水返佣(点差 + 管理费)+ 月度净输赢分红;入**佣金余额** |
| **返水** | **玩家侧**实时返水(PRD-018),入玩家**现金余额**;与返佣不同 |
| **流水返佣 / 提成** | 按点差 / 管理费**实时**发放的返佣 |
| **净输赢** | 平台对某下级的净盈利(可正可负,正 = 平台赢);原「负盈利」统一为此 |
| **净输赢分红** | 月底按 (本月净输赢 + 上月结余) × 分红比例 计的合伙人分红 |
| **平台费 / 资金费 / 扶持红利 / 活动优惠** | 计入直属上级、从返佣中扣减的成本(§7.1) |
| **有效会员 / 有效新增** | 见 §6.2(存款 > 500,或 存款 > 100 且 流水 > 1000;新增追加当周期注册) |
| **上月结余** | 跨月结转的净输赢余额(可为负,亏损结转) |

## 2. 身份与层级

### 2.1 两种身份(无等级)

| 身份 | 说明 | 分佣深度 |
| --- | --- | --- |
| **合伙人** | 普通推广节点。每个玩家 ID 即邀请码,可邀请下级建立层级 | 沿向上**最多 2 级合伙人**(直属上级、上上级)参与点差返佣 |
| **高级合伙人** | 平台直招的顶层节点,其名下**无限级**下级的返佣最终追溯至该高级合伙人 | 绑定关系下**无限层**(`senior_partner_id`) |

> **取消等级**:V2 不再有 S0-SSS / G1-GGG 等级与按等级递增的返佣比例。返佣比例由**每个合伙人独立的点位配置**决定(见 §3)。

### 2.2 身份获取规则

* **玩家 → 合伙人**:**管理员在后台可将任意玩家提升为合伙人**,**并在此时设置其点位配置**(见 §3)。
* **玩家 / 合伙人 → 高级合伙人**:**只能通过两种途径**——
  1. 管理员在控制台**直接创建 / 指定**;
  2. 对**没有上级(无 `inviter_id`)的玩家**执行提升。
  > 即:**有上级的玩家不能成为高级合伙人**(避免破坏既有追溯链)。
* **点位配置**:高级合伙人同样有自己的点位配置,**创建时设置、管理员可随时修改**(见 §3)。

### 2.3 邀请关系与绑定记录

每个玩家注册 / 被邀请时,系统**固化记录三类向上关系**,后续不随对方身份变化而漂移:

| 字段 | 含义 |
| --- | --- |
| **上级**(`inviter_id`) | 直接邀请人(直属上级) |
| **上上级**(`inviter_id_2`,冗余) | 直属上级的直属上级(第 2 级合伙人)。**冗余存储在玩家信息中、紧邻 `inviter_id`**,免去向上递归查父级,加速 2 级分红 / 点差计算 |
| **高级合伙人**(`senior_partner_id`) | 该链路最终归属的高级合伙人(可为空,非强制) |

* 关系一经绑定不可篡改;管理员调整归属属特殊运维操作,需审计。
* **非强制高级合伙人**:不是所有玩家都有高级合伙人作为最终追溯对象。

## 3. 点位配置

### 3.1 每个合伙人独立配置

* **每个合伙人 / 高级合伙人都有自己的一套点位配置**,**在管理员「添加 / 提升 / 创建」时设置初始点位**;**管理员后续可随时修改**(调整对调整后的新投注生效)。
* 点位维度 = **游戏类型**,口径**复用 PRD-018 §5.3**:**PG电子 / 其他电子 / 棋牌自由桌 / 棋牌自建房 / 预测市场**(后续游戏类型新增时同步扩展)。**点位按游戏(类型)独立区分,不区分自有 / 第三方。**
* 每个游戏类型配一个**流水返点点位(%)**,即该合伙人对该类游戏流水的自身返点比例。

**示例配置(草稿,后台可配):**

| 游戏类型 | 流水返点点位 |
| --- | --- |
| PG电子 | 0.5% |
| 其他电子 | 0.5% |
| 棋牌自由桌 | 0.5% |
| 棋牌自建房 | 0.3% |
| 预测市场 | 0.3% |

### 3.2 点差与管理费

返佣**向上逐级追溯**,**上级只能拿到「点差」**:

```
某级合伙人的点差返佣比例 = 本级该游戏类型点位 − 直接下级该游戏类型点位
```

* **上级点位 > 下级点位(点差为正)**:上级按点差 × 下级该类游戏流水获得返佣。
* **上级点位 ≤ 下级点位(点位相同或更低)**:上级拿不到点差,**一律改为适用「管理费」**——上级获得**下级该笔返佣的 10%**(**默认 10%,全局可配;不按游戏类型区分**),**由平台额外补贴,不从下级收益中扣除**;点差为负不倒扣。
* 点差 / 管理费**逐级独立计算**:沿链路每相邻两级各算一次,直至高级合伙人。

> **示例**:下级 A(预测市场点位 0.3%)产生预测市场流水 10,000;直属上级 B 点位 0.5% → B 点差 = 0.2% → B 返佣 = 10,000 × 0.2% = 20。若 B 点位也是 0.3%(相同)→ B 改拿 A 该笔返佣的 10% 管理费(平台补贴)。

## 4. 返佣总览

返佣**包含两部分**,分别独立结算:

| 部分 | 参与游戏 | 基数 | 结算时机 |
| --- | --- | --- | --- |
| **流水返佣** | **所有游戏** | 下级有效投注流水(现金场次) | **实时返佣**,投注结算后即时计算入账 |
| **净输赢分红** | **仅部分游戏**(后台按场馆配置参与) | 直属下级的**净输赢**(平台对该下级的净盈利) | **月底计算**,按有效会员 / 新增考核分红 |

* 两部分均**排除场馆彩金场次**流水与盈亏(PRD-013)。
* 两部分**都按身份链路向上分配**(合伙人最多 2 级 + 高级合伙人无限级)。

## 5. 流水返佣(实时)

### 5.1 计算

* 触发:下级玩家每笔有效投注结算后,按 **§3 点差 / 管理费**实时计算上链各级返佣并入账佣金账户。
* 基数:该下级在该游戏类型的**现金场次有效投注流水**(场馆彩金排除)。
* 玩家自身:玩家本人按**自身点位**获得一级返点(归入自己佣金账户)。
* **返佣金额构成**:合伙人「返佣」= **点差返佣 + 管理费**(§3.2)——二者都是实时返佣、都实时入佣金账户,统计与月结的「提成金额」均含管理费。

### 5.2 与玩家实时返水的区别

* **玩家实时返水**(PRD-018 §5.3)= 玩家对自己投注的返水,无需流水可直接提现。
* **流水返佣**(本节)= **上级**对**下级**流水按点差获得的推广分润,入**佣金账户**,转出受 §7 约束。

## 6. 净输赢分红(月结)

> **术语统一**:本系统「负盈利」与「净输赢」是同一概念,统一用**净输赢**(平台视角净盈利,可正可负,正=平台赢)。

### 6.1 净输赢基数

* 以**直属下级**为单位,统计**当月**平台对该下级的**净输赢**(即玩家亏损 = 平台净赢)。
* **仅在 §7.1.1 游戏独立配置中勾选「参与净输赢分红」的游戏**计入(默认参与范围待运营配置)。
* 场馆彩金场次盈亏排除(PRD-013):场馆彩金场次玩家亏损不计入净输赢基数。

### 6.2 分红比例(双门槛考核)

按合伙人**当月**的「有效会员」与「有效新增」考核,**需同时满足某档的两个门槛,取最高达标档**的分红比例:

| 有效会员 | 有效新增 | 净输赢分红比例 |
| --- | --- | --- |
| 5 | 0 | 50% |
| 5 | 5 | 55% |
| 10 | 10 | 60% |
| 25 | 25 | 70% |
| 50 | 50 | 75% |

* **不足 5 有效会员 → 0%**,不参与净输赢分红。
* 取档规则:同时满足「有效会员 ≥ 该行」且「有效新增 ≥ 该行」才算达标该行;取所有达标行中**分红比例最高**的一档。
* **有效会员 / 有效新增(全平台统一口径)**:
  * **有效会员** = **某周期内**满足任一条件:① **累计存款 > 500**;**或** ② **累计存款 > 100 且 流水 > 1000**。
    > 例:本月有效会员 = 本自然月内,累计存款 > 500,**或**(累计存款 > 100 且 流水 > 1000)。
  * **有效新增会员** = 在「有效会员」基础上,**追加「该周期内新注册」条件**。
  * **统一口径**:本定义为**全平台统一**界定——合伙人系统、邀请活动等所有用到「有效会员」的模块**采用同一口径**(门槛 500 / 100 / 1000 后台可配)。
  * ⚠️ **与现状差异(需统一改造)**:现网各处口径不一——合伙人按**流水 ≥ `partner_valid_member_threshold`(默认 500)**、邀请活动按**流水达标(默认 10 万、终身窗口)**、首存为二元判断。本 PRD 改为**上述以存款为主的统一口径**,实施需把各模块收口为同一逻辑(见 §9 数据结构)。

### 6.3 月结算与可为负

* **结算周期**:每月一次,UTC+0 月末跑批。
* **正盈利直接扣除**:若某直属下级当月对平台为**正盈利**(平台亏钱 / 玩家赢),该部分**直接从该合伙人当月净输赢中扣除**,**净输赢 / 佣金可以为负数**(区别于 PRD-014 净佣金以 0 为底)。

### 6.4 收益构成与月度净输赢分红公式(权威)

合伙人收益由**两部分相加**构成,二者**互不重复**:

1. **实时提成(返佣)** —— §5 流水返佣 / 点差,**投注结算后实时发放**,即「返佣部分」。
2. **月度净输赢分红** —— 月底按下式计算。

```
本月净输赢 = 总输赢 − 总优惠活动 − 返水金额 − 提成金额 − 代理红利 − 邀请好友 − 平台费 − 资金费

月度净输赢分红 = (本月净输赢 + 上月结余) × 净输赢分红比例

合伙人当月总收益 = 实时提成(返佣) + 月度净输赢分红
```

**字段对应:**

| 项 | 含义 | 来源 |
| --- | --- | --- |
| 总输赢 | 团队所有游戏平台视角总输赢(正=平台赢) | §8.5 |
| 总优惠活动 | 活动优惠成本(充值送等) | §7.1 活动优惠 |
| 返水金额 | 下级玩家实时返水合计 | PRD-018 §5.3 |
| 提成金额 | **当月已实时发放的流水返佣 = 点差返佣 + 管理费**;在净输赢中减去,**避免与净输赢分红重复计** | §5 |
| 代理红利 | **扶持红利**(下级周俸禄 / 升级礼) | §7.1 扶持红利 |
| 邀请好友 | 邀请好友奖励成本 | 邀请活动 |
| 平台费 | **Σ(各游戏下级流水 × 各游戏平台费率)**(场馆费即平台费) | §7.1.1 |
| 资金费 | 充值 / 提现渠道资金费 | §7.1 资金费 |
| 上月结余 | 上月结转的净输赢余额(**可为负**) | 月度结转 |
| 净输赢分红比例 | §6.2 双门槛考核(有效会员 / 有效新增)对应比例 | §6.2 |

**要点:**

* **两部分独立**:实时提成(返佣)按 §5 实时发放;月度净输赢分红按上式月结。净输赢里减「提成金额」即扣掉本月已实时发的返佣,**两者不重复**。
* **净输赢分红比例作用于(本月净输赢 + 上月结余)整体** → **本式为准**,取代 §6.3 早期的「净输赢分红 = 基数 × 比例」简化式。
* **结转规则(亏损结转、盈利清零)**:令 `runningNet = 本月净输赢 + 上月结余`。
  * `runningNet ≤ 0` → **当月净输赢分红 = 0**(不向代理倒扣现金),**结转下月「上月结余」= runningNet**(负数带入下月);
  * `runningNet > 0` → **净输赢分红 = runningNet × 比例**,**结转下月 = 0**(盈利不结转,(1 − 比例)为平台留存)。
  * 即**亏损跨月累计、直至被后续盈利填平;盈利月结清不带入**。所谓「返佣可为负」指**台账结余**可为负并结转,**实付分红不为负**。
* **费用每月归零**:总输赢 / 优惠 / 返水 / 提成 / 代理红利 / 邀请 / 平台费 / 资金费 都是**本月**量、**每月从 0 重新累计**(平台费 / 资金费已在当月净输赢里扣过、不跨月);**唯一跨月的是(负的)净输赢结余**。
* **平台费 = 流水 × 费率**(非总输赢);「场馆费」与「平台费」为同一项,统一用「平台费」。

## 7. 费用扣减、佣金账户与转出

### 7.1 直属上级承担的成本

下级产生的以下成本,**全部计入「直属上级」**,并**从其佣金中扣除**:

| 成本项 | 来源 / 计算 | 说明 |
| --- | --- | --- |
| **平台费** | 下级**流水 × 平台费率**(费率**按供应商 / 游戏独立配置**,见 §7.1.1) | 直属上级在获得返佣的同时,为下级流水向平台支付平台费;**费率因供应商而异,非统一 7%** |
| **资金费** | **资金费率由各充值 / 提现渠道独立配置**(默认值待运营给出) | ① 玩家**提现免手续费**时,由代理提供资金费;② 玩家**充值玩家免费**,按**充值渠道**由代理提供资金费。各渠道费率独立配置 |
| **扶持红利** | 下级玩家领取的**周俸禄 / 升级礼**(PRD-018) | 下级玩家获得的玩家成长权益,计入直属上级的「红利」成本 |
| **活动优惠** | **充值送**等活动赠送 | 下级享受的活动赠送由「活动优惠」承担,计入直属上级 |

#### 7.1.1 游戏独立配置(平台费率 + 是否参与净输赢分红)

平台按**供应商 / 游戏维度**维护一套「**游戏独立配置**」,每个游戏含两项:**① 平台费率、② 是否参与净输赢分红**。

各供应商 / 游戏**配置示例**(① 平台费率、② 是否参与净输赢分红):

| 供应商 / 游戏 | 平台费率 | 参与净输赢分红 |
| --- | --- | --- |
| CQ9 | 7.50% | ✓ |
| EVOPLAY | 7% | ✓ |

### 7.2 扣减与清零

* **平台费 / 资金费 / 扶持红利(代理红利)/ 活动优惠 均作为 §6.4 「本月净输赢」的扣减项**,从代理收益中扣除。
* **月度计数清零、净输赢结余跨月**:各成本 / 提成 / 返水的**当月计数月末清零、不跨月累计**;但**净输赢结余按 §6.4 跨月结转**(可为负,亏损带入下月「上月结余」)。
* **月度净输赢分红以 §6.4 公式为准**(`月度净输赢分红 =(本月净输赢 + 上月结余)× 净输赢分红比例`)。

### 7.3 佣金账户与现金账户

* 沿用账户体系:**佣金余额** 与 **现金余额** 分离(PRD-012)。
* **返佣(合伙人)→ 佣金余额**:流水返佣(提成)与净输赢分红均入**佣金余额**,成本扣减在佣金侧核算,转出现金受 §7.4 约束。
* **返水(玩家)→ 现金余额**:玩家实时返水**直接到账现金余额**(非佣金账户),无需流水可直接提现(PRD-018 §5.3)。

### 7.4 转出规则(佣金 → 现金)

* **受 PRD-012 `commission_lock` 约束**:`commission_lock = 开` 时,佣金余额**禁止**转出到现金余额(弹窗联系客服)。
* **未受 `commission_lock` 控制时**:佣金账户中**超出(平台费 + 资金费 + 扶持红利 + 活动优惠)应计成本的部分**,可转出到现金余额。
* 即:可转出额 = max(0, 佣金余额 − 当期应计(平台费 + 资金费 + 扶持红利 + 活动优惠))。
* **返水 / 提成不计入转出保留**:返水是平台付给玩家的成本(已在 §6.4 月度净输赢分红中扣减一次),提成是代理已实时入账的应得佣金;二者均非「代理欠平台」的应计成本,**不进保留清单**(避免重复扣)。
* **负的上月结余不影响可转出额**:负结余只**压制未来净输赢分红**,不倒扣已入账佣金。
* 转出路径沿用 PRD-012:**不计入 AML 分段流水锁**,视为可直接提现。

## 10. 场景走查

### A:实时返佣 —— 点差 + 管理费(单笔)

下级玩家 A 在预测市场产生有效投注流水 10,000;链路 A → B(上级)→ C(上上级)。点位:A 0.3%、B 0.5%、C 0.5%。

| 层级 | 点位 | 相对下级 | 所得(实时入佣金余额) |
| --- | --- | --- | --- |
| A(投注玩家) | 0.3% | 自身一级返点 | 10,000 × 0.3% = **30** |
| B(直属上级) | 0.5% | 点差 +0.2% | 10,000 × 0.2% = **20** |
| C(上上级) | 0.5% | = B(点位相同) | 管理费 = B 返佣 × 10% = **2**(平台补贴) |

### B:月度净输赢分红 —— 亏损结转 / 盈利清零(分红比例示例 60%)

| 月 | 本月净输赢 | 上月结余 | runningNet | 实付分红 | 结转下月 |
| --- | --- | --- | --- | --- | --- |
| 1 | +10,000 | 0 | +10,000 | 6,000 | 0 |
| 2 | −8,000 | 0 | −8,000 | **0** | −8,000 |
| 3 | +5,000 | −8,000 | −3,000 | **0** | −3,000 |
| 4 | +10,000 | −3,000 | +7,000 | 4,200 | 0 |

→ 亏损跨月累计直至被盈利填平;实付分红不为负;实时返佣(提成)各月照发、不受影响。

### C:费用扣减与佣金转出(某月,commission_lock = 关)

| 项 | 金额 |
| --- | --- |
| 佣金余额(实时返佣已入) | 1,000 |
| 当期应计:平台费 300 + 资金费 50 + 扶持红利 100 + 活动优惠 50 | 500 |
| 可转出 = max(0, 1,000 − 500) | **500** → 转入现金余额(不计 AML 锁) |

> 返水 / 提成不计入转出保留;若 `commission_lock = 开`,则全额禁止转出(弹窗联系客服)。

## 11. 验收标准

**身份与层级**

- [ ] 合伙人 / 高级合伙人**无等级**;管理员可提升玩家为合伙人并**同时设点位**;高级合伙人仅「控制台创建」或「无上级玩家提升」
- [ ] 记录 `inviter_id`(上级)/ `inviter_id_2`(上上级冗余)/ `senior_partner_id`(高级合伙人);绑定不可篡改

**点位与实时返佣**

- [ ] 每个合伙人 / 高级合伙人**独立点位**(按游戏类型,§3),创建时设、管理员可改
- [ ] 上级点位 > 下级 → **点差返佣**;上级点位 **≤** 下级 → **管理费**(下级该笔返佣 10%、平台补贴、全局可配、不按游戏区分)
- [ ] 返佣 = **点差 + 管理费**,实时入**佣金余额**;场馆彩金场次排除

**净输赢分红(月结)**

- [ ] 仅「游戏独立配置」勾选**参与净输赢分红**的游戏计入净输赢基数(未配置默认 ✗)
- [ ] 分红比例按 §6.2 **双门槛**(有效会员 + 有效新增)取最高达标档;< 5 有效会员 = 0%
- [ ] 月度净输赢分红 = (本月净输赢 + 上月结余) × 比例;**亏损结转、盈利清零、实付分红不为负**
- [ ] 有效会员 = 存款 > 500,或 存款 > 100 且 流水 > 1000;有效新增追加「当周期注册」;按**当月**窗口

**费用与账户**

- [ ] 平台费 = Σ(各游戏流水 × 费率)(未配置 = 其他 7%);资金费按渠道配置;扶持红利 = 下级周俸禄 / 升级礼;活动优惠
- [ ] 费用 / 提成 / 返水**每月归零**,唯一跨月 = 负的净输赢结余
- [ ] **返佣 → 佣金余额、返水 → 现金余额**
- [ ] `commission_lock = 开` 禁止转出;关时可转出 = max(0, 佣金余额 − 应计(平台费 + 资金费 + 扶持红利 + 活动优惠));**返水 / 提成不计入保留**;转出**不计 AML 锁**

**数据看板**

- [ ] **全部 / 直属 / 团队** 切换;下级概览(成员 / 首存 / 注册 / 有效)
- [ ] 本月概览各字段齐全(含**返佣金额、返水金额分列**;平台费 / 资金费 / 扶持红利 / 活动优惠;总输赢 + 净输赢单列)
- [ ] 分游戏报表(总输赢 / 平台费率 / 平台费用)
- [ ] 直属下级列表 + 团队列表;下级**充值 / 提现 / 流水**记录(复用已有 3 接口,按层级隔离)
