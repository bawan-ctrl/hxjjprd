# PRD-011: IP白名单管理系统

**状态：** 定稿  
**日期：** 2026-05-14（bawan）  
**优先级：** P2  
 

## 1. 需求概述

通过IP白名单机制控制管理后台访问权限，只有名单内的IP才能访问后台。支持增删改查、搜索、备注等功能，方便运营、后台使用人员日常管理和安全性。

## 4. 功能需求

### 4.1 IP白名单开关

**位置：** 页面顶部，标题下方

‌

**开关状态：**

* **开启：** 只有白名单内的IP可以访问后台
* **关闭：** 所有IP都可以访问后台（默认状态）

**使用场景：**

* 测试环境平常关闭，不限制访问
* 生产环境有需要时开启，提高安全性
* 临时需要开放访问时可以快速关闭

**切换规则：**

* 点击开关立即生效
* 开启时显示提示："⚠️ 开启后只有白名单内的IP可访问，请确认您的IP已在名单中"
* 关闭时显示提示："IP白名单已关闭，所有IP均可访问"

**特别说明：**

* 开关状态独立于白名单列表
* 关闭开关时，白名单列表仍然保留，不会被清空
* 再次开启时，使用之前配置的白名单列表

---

### 4.2 添加IP

**入口：** 点击页面左上角"+ 添加IP"按钮

**弹窗表单：**

1. **IP地址（必填）**

    * 支持IPv4格式：`192.168.1.1`
    * 支持IPv6格式：`2001:0db8::1`
    * 一次只能添加一个IP
    
2. **备注（必填）**

    * 用于说明IP的用途
    * 例如："CEO办公室"、"开发环境"、"测试服务器"
    

**校验规则：**

* IP地址不能为空
* IP地址格式必须正确（IPv4或IPv6）
* IP地址不能重复
* 备注不能为空

**保存后：**

* 关闭弹窗
* 提示"添加成功"
* 列表自动刷新，新IP显示在顶部
* 自动记录更新时间和操作人

---

### 4.3 搜索功能

**使用方式：**

* 在搜索框输入关键词
* 点击"查询"按钮触发搜索

**搜索范围：**

* IP地址（模糊匹配）
* 备注内容（模糊匹配）

**搜索结果：**

* 显示匹配的记录
* 分页信息自动更新
* 无结果时显示"未找到匹配的记录"
* 清空搜索框后点查询，恢复显示全部

**示例：**

* 搜索"192" → 显示所有包含192的IP
* 搜索"办公室" → 显示备注中包含"办公室"的记录

---

### 4.5 编辑功能

**入口：** 点击表格行右侧的"编辑"按钮

**弹窗表单：**

* 预填充当前IP地址和备注
* 可修改IP地址或备注
* 校验规则与添加功能相同

**保存后：**

* 关闭弹窗
* 提示"更新成功"
* 列表中对应记录更新
* 更新时间和操作人自动更新

---

### 4.6 删除功能

**入口：** 点击表格行右侧的"删除"按钮

**确认提示：**

```
确定要删除IP「192.168.1.1」吗？

此操作不可恢复！
```

**删除后：**

* 提示"删除成功"
* 从列表中移除该记录
* 自动刷新列表

## 6. 业务规则

### 6.1 开关控制

* **默认状态：** 关闭（所有IP都可访问）
* **开启后：** 只有白名单内的IP可以访问后台
* **切换生效：** 立即生效，无需等待
* **开关独立：** 开关状态与白名单列表独立，关闭开关不会清空列表

### 6.2 IP格式要求

* **IPv4/IPv6** 
* 一次只能添加一个IP地址
* IP地址不能重复

### 6.3 访问控制逻辑

1. 用户访问后台时，首先检查IP白名单开关
2. **开关关闭** → 允许所有IP访问
3. **开关开启** → 检查访问者IP是否在白名单中

    * IP在白名单内 → 允许访问
    * IP不在白名单内 → 返回403拒绝访问
    

### 6.4 安全提醒

* 开启开关前，建议先确认自己的IP已在白名单中
* 系统会在开启时弹窗提醒："⚠️ 开启后只有白名单内的IP可访问，请确认您的IP已在名单中"等友好话语

## 7. 异常处理

| 场景 | 处理方式 |
| --- | --- |
| IP地址为空 | 显示错误提示："请输入IP地址" |
| IP格式错误 | 显示错误提示："IP地址格式不正确" |
| IP地址重复 | 显示错误提示："该IP地址已存在" |
| 备注为空 | 显示错误提示："请输入备注" |
| 搜索无结果 | 显示"未找到匹配的记录" |
| 列表为空 | 显示"暂无数据，点击'添加IP'按钮添加第一条记录" |

## 9. 权限控制

**权限逻辑：** 全有或全无

* **有权限：** 可以看到"IP白名单管理"菜单，可以使用所有功能（增删改查）
* **无权限：** 看不到"IP白名单管理"菜单，无法访问该页面

**权限标识：** `system:ip_whitelist`

---

## 10. 运营使用建议

* **测试环境：** 建议平常保持开关关闭，不影响测试
* **生产环境：** 根据实际需要决定是否开启
* **首次配置：** 建议先添加自己的IP，测试通过后再开启开关
* **备注规范：** 建议写清楚IP归属人或用途，方便后续查找 
* **定期维护：** 定期清理不再使用的IP，保持列表整洁
