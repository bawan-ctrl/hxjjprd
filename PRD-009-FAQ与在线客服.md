# PRD-009:FAQ 与在线客服

**版本**:V1.0(bawan) 

**整理日期**:2026-05-12

**状态：** 已完成

---

## 1. 文档概览

* **产品名称**:HashRace Platform 常见问题(FAQ)与在线客服入口
* **功能名称**:FAQ 主页、搜索、模块分类、问题详情、在线客服入口、后台内容管理
* **目标**:为用户提供自助服务首个入口,降低客服压力,并在用户无法自助解决问题时无缝过渡到人工客服

## 3. 核心功能需求

### 3.1 常见问题(FAQ)主页

FAQ 主页是用户自助服务的首个入口,设计应简洁直观。

* **搜索功能区**:页面顶部必须配置一个**醒目的搜索框**,占位符应清晰提示用户"搜索你想要查找的问题"
* **推荐问题列表**:搜索框下方展示一组精选的**最常见或当前热议**的问题,用户可以直接点击查看解答
* **按模块分类导航**:主页核心内容,将所有问题按业务模块进行清晰分类展示(静态展示,无需接口),例如:**锦标赛业务、竞猜业务、充值业务、账号业务**等
* **在线客服入口**:在页面底部设置一个**颜色醒目、始终可见**的"在线客服"按钮,作为用户最终无法自助解决问题时的求助渠道

### 3.2 搜索功能与历史记录

系统需提供高效的搜索能力,并优化用户体验。

* **搜索匹配机制**:当用户输入关键词时,系统应根据关键词对问题的**标题和内容**进行**模糊匹配**
* **搜索结果页**:匹配到的相关问题应清晰列表展示,并可根据相关度或所属模块进行排序
* **无结果处理**:当搜索结果为空时,系统必须展示\*\*"没有找到相关回答"\*\* 的友好页面,并在该页面保留\*\*"在线客服"\*\* 入口,引导用户寻求人工帮助
* **搜索历史记录**:搜索框下方或点击搜索框时,应展示用户最近搜索过的 **5-10 条关键词或问题记录**(前端缓存),方便用户快速重复搜索

### 3.3 模块分类列表与问题详情

用户点击主页的分类模块后,应进入该模块的问题列表。

* **模块切换导航**:列表页顶部应支持用户按业务模块快速切换,必须包含:**账号业务、竞猜业务、锦标赛业务、充值业务、提现业务**等主要模块(**动态获取业务类型**)
* **问题列表展示**:清晰展示当前选中模块下的所有问题,可按后台配置的优先级或发布时间排序
* **问题详情页**:点击任意问题,用户将跳转至该问题的详细解答页面。解答内容应支持**富文本**等多种格式,以确保内容清晰易懂

## 4. 后台管理需求

### 4.1 内容编辑与管理

管理员需要一个可视化的后台来维护和更新 FAQ 内容。

* **问题内容编辑**:需提供**可视化编辑器**(富文本),用于编辑问题标题和详细解答内容,保证内容的丰富性和格式的可读性
* **模块分类管理**:管理员可灵活**自定义业务模块分类**,并为每个问题指定所属模块
* **推荐问题配置**:可配置主页\*\*"推荐问题"\*\* 列表,管理员手动指定等方式进行筛选和权重排序

### 4.2 搜索优化与审计

* **关键词/标签管理**:必须为每个问题配置**多个关键词/标签**,以大幅优化搜索结果的准确性和匹配度
* **操作审计**:记录问题内容的**新增、修改、删除**等所有操作,包括**操作员 ID 和时间**,以保证内容准确性和可追溯性
