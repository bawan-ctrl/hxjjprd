# PRD-003:优惠券中心

**版本**:V1.0(bawan) 

**整理日期**:2026-05-12

**状态：** 已完成

---

## 1. 文档概览

* **产品名称**:HashRace Platform 优惠券中心
* **功能名称**:优惠券(凭证)展示、管理、兑换入口、后台凭证池配置
* **项目目标**:

    * **统一展示**:为用户提供一个清晰、集中的优惠券和奖励凭证管理界面
    * **明确状态**:清楚区分和展示凭证的「可用」和「不可用」状态
    * **强化转化**:通过「兑换优惠券」入口,整合奖励码兑换功能
    
* **项目背景**:平台需要一个标准化的前端模块来管理和展示用户通过**邀请活动、兑换中心、系统发放**等渠道获得的各类奖励凭证

## 3. 核心功能需求

### 3.1 优惠券中心主界面

#### 3.1.1 状态筛选与切换

顶部必须包含两个清晰的 Tab 按钮,用于对凭证状态进行筛选:

* **「可用」**:展示所有当前有效且未使用的凭证
* **「不可用」**:展示所有已使用、已失效或已撤销的凭证

#### 3.1.2 优惠券列表展示

* 所有凭证应以**卡片形式**列表展示。列表内容会根据用户当前选择的 Tab 实时刷新
* 每张卡片必须醒目地展示凭证的**名称、面额/价值、有效期**和**使用规则**等

#### 3.1.3 兑换功能入口

* 在主界面的底部或易于发现的区域,必须设置一个\*\*「兑换优惠券」\*\*的入口,点击后跳转至兑换中心功能

#### 3.1.4 无数据状态

* 当任一 Tab 下无凭证时,应显示友好的\*\*「暂无优惠券」\*\*提示

### 3.2 优惠券卡片详情展示

每张优惠券卡片必须清晰展示以下信息:

* **凭证名称**:简洁描述优惠券用途(例:开年大吉充值礼)
* **有效期**:明确的截止日期和时间(例:有效期至 2025-09-30 23:59:59)
* **面额/价值**:醒目展示该券的价值\*\*(面额+单位)\*\*(例:0.5%)
* **小标题/描述**:充值立享 0.5% 额外红利
* **使用范围/规则**:提示该券的使用规则

### 3.3 Tab 状态与逻辑

| Tab 名称 | 逻辑定义 | 包含的底层状态 | 样式要求 |
| --- | --- | --- | --- |
| **可用** | 凭证在**有效期内**且**未使用** | 对应原"可使用"状态 | 正常颜色,醒目突出 |
| **不可用** | 凭证**已使用**或**已失效/已撤销** | 对应原"已使用"和"已失效"状态的集合 | 灰色或淡化展示,显示具体原因(已使用或已失效) |

## 4. 管理后台需求

### 4.1 凭证池配置与管理

管理员需要一个配置中心来灵活定义和管理不同类型的奖励凭证及其规则。

#### 4.1.1 凭证类型与规则定义

系统支持以下核心凭证类型,每种类型都有独立的规则和可配置参数:

| 凭证类型 | 规则模板(可编辑) |
| --- | --- |
| **百分比红利充值券** | 用户充值时,按充值金额的固定百分比获得红利/泥码。红利需满足特定 AML 流水才能提现 |
| **固定额度红利充值券** | 用户充值满足**最低门槛**后,可获得固定额度的红利/泥码。红利需满足特定 AML 流水才能提现 |
| **免提现费券** | 用户提现时,可抵扣一笔提现操作的服务费用(按次或按额度) |
| **专属赛事报名券** | 用户可免费报名参加指定的某一场或某一类锦标赛。每张券对应一场比赛的报名费 |

#### 4.1.2 凭证的通用与专属参数

每张优惠券在后台配置时,都必须包含以下详细信息:

**凭证通用属性(所有券共有)**:

* **凭证类型**:百分比红利券(下拉选择:百分比红利券、固定额度红利充值券、免提现费券、新手赛报名券等)
* **凭证参数**:(0.5, 88, 1, 40)
* **凭证名称**:开年大吉充值礼
* **小标题/描述**:充值立享 0.5% 额外红利
* **有效期**:明确的截止日期和时间(例如:**2026-03-03 00:00:00**)
* **面额/数值**(仅展示):凭证的主要价值数值(例如:0.5, 10)
* **单位**:凭证价值的单位(例如:%, 元, 次, 张)

**凭证专属参数(根据类型配置)**:

| 凭证类型 | 专属可配置参数 |
| --- | --- |
| **百分比红利充值券** | **加赠百分比**(例:0.5%)、**加赠上限**(例:88 元)、**AML 流水倍数**(例:X=1, Y=40)、**支持 USDT 充值**(是/否)、**支持普通货币充值**(是/否) |
| **固定额度红利充值券** | **最低充值门槛**(例:1000 元)、**固定赠送额度**(例:10 元)、**AML 流水倍数**(例:X=1, Y=40)、**支持 USDT 充值**(是/否)、**支持普通货币充值**(是/否) |
| **免提现费券** | **抵扣类型**(按次/按额)、**抵扣额度**(若按额度) |
| **专属赛事报名券** | **适用赛事 ID**(例:新手赛、200pk 场、50000PK 场、封神赛之一夜暴富报名券、封神赛之群雄逐鹿等由 HR Gaming 提供)或**赛事类型**(例:经典斗地主、德州扑克) |

### 4.2 玩家优惠券管理界面

管理员需要一个界面来查看和管理所有玩家的优惠券持有情况,并能进行发放。

1. **玩家查询**:支持通过玩家 ID 或用户名精确查询其**当前持有的优惠券列表**
2. **优惠券列表展示**:

    * **筛选与排序**:支持按优惠券**类型、状态**(可用/不可用)、**有效期**进行筛选和排序
    * **展示字段**:列表应清晰展示每张优惠券的**类型、凭证参数、名称、有效期、当前状态**,以及关键的**面额(面额+单位)**
    
3. **发放操作(快捷入口)**:在玩家的优惠券列表界面,应提供一个\*\*「发放优惠券」\*\*的快捷按钮,允许管理员快速选择凭证类型并向该玩家发放指定数量的凭证(需记录操作日志),**支持批量发送**
4. **撤销操作**:管理员可对玩家账户中\*\*「可用」**状态的优惠券进行**撤销\*\*操作(凭证状态变更为「已撤销」)
